# Frontend Design Skill

Build production-grade frontend components and applications with distinctive, intentional design that avoids generic AI aesthetics.

## Design Methodology

Start by understanding context — purpose, target audience, and technical constraints — then commit to a bold aesthetic direction (minimalist, maximalist, retro-futuristic, brutalist, etc.).

## Key Implementation Principles

- **Typography**: Choose fonts that are beautiful, unique, and interesting. Avoid generic fonts like Arial and Inter in favor of distinctive, characterful options.
- **Color & Theme**: Develop cohesive palettes with dominant colors and sharp accents. Use CSS custom properties for consistency.
- **Motion**: Prioritize impactful animations, particularly page-load sequences and scroll-triggered effects.
- **Layout**: Embrace asymmetry, overlap, and unexpected compositions rather than predictable grids.
- **Details**: Add atmospheric elements like textures, gradients, and custom effects that elevate the design.

## What to Avoid

- Generic design patterns
- Overused font families (Arial, Inter, Roboto)
- Clichéd color schemes (especially purple gradients)
- Cookie-cutter layouts lacking context-specific character
- "AI slop" aesthetics — designs that look auto-generated

## Core Philosophy

Match implementation complexity to the aesthetic vision, whether that's elaborate maximalism or refined minimalism. The goal is intentional, memorable design that feels genuinely crafted for its specific purpose.

Always ask: what does this page *feel* like? What emotion should it evoke? Then build toward that answer.
